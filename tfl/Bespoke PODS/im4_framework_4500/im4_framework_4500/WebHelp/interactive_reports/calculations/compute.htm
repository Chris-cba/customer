Compute

The Compute option allows you to add a “derived” or computed column to your report Pod. These new columns can be the result of mathematical computations, e.g. Target Date minus Completion Date, or standard Oracle functions applied to existing columns, e.g. INITCAP, LTRIM, TO_DATE etc. The figure below shows a simple example that performs a calculation to compute the number of days late based upon subtracting one date from another.

How to create a new Computed Column

From the computation field select “- New Computation –”

Enter a name for the new Report Column in the “Column Heading” field

Select a Format mask from the list of values to be applied against the column, e.g. 5,210, 5,210.10, 01-JAN-2000, etc

Enter the calculation to be performed in the Computation field. This can either be manually entered or you can use the Columns, Keypad and Functions area which simply provide shortcuts for commonly used Keys.

Within the Computation field existing Report Columns are referred to by their Alias name as displayed in the Column List.

The display sequence of the new computed Column can be set using the “Select Columns ” option in the Actions menu. Computed Columns are indicated in the list by **

How to delete a Computed Column

From the computation field select the required value from the List of values

Press the [Delete] button. This will remove the computed column from the Interactive Report Pod.
